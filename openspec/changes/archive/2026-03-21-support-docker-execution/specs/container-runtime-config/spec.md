## ADDED Requirements

### Requirement: Environment Variable Injection
The Docker container SHALL accept environment variables for all Relay Service configurations, including platform tokens and provider API keys.

#### Scenario: Pass Environment Variables
- **WHEN** the user provides `DISCORD_TOKEN` as an environment variable to the container
- **THEN** the system inside the container SHALL correctly initialize the Discord platform with that token.

### Requirement: External Configuration Mounting
The system SHALL support mounting a `.env` file from the host machine to provide configuration without rebuilds.

#### Scenario: Use External .env File
- **WHEN** a `.env` file is mounted to `/app/.env` in the container
- **THEN** the `relay-service` SHALL read configuration from that mounted file.

### Requirement: Persistence Support
The container environment SHALL support persistent data storage through volume mounting if necessary (e.g., for local logs or data files).

#### Scenario: Persist Logs via Volume
- **WHEN** a volume is mounted to the container's log directory
- **THEN** logs generated by the service SHALL be visible on the host machine.
